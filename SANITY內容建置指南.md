# brand-web × studio-medmanager 假資料建置指南

> 目的：目前 `studio-medmanager`（Sanity 後台）資料庫**沒有任何資料**，所以 brand-web 形象站每個欄位都顯示「**-**」。
> 本文件逐欄列出「**該在哪個欄位填什麼**」，照著建立並 **Publish（發布）** 後，畫面就會帶出內容。
>
> - 對應後台：`studio-medmanager`，`projectId: qrepjm4m`，`dataset: production`
> - 對應前端：`brand-web`（Vue 3 + Vite），資料來源 `src/sanity/queries.js`
> - 設計原則：**沒資料就顯示「-」，不放假資料**。下方「建議填入值」即原本的示意文案，提供你直接複製。

---

## 0. 開始之前

1. **確認連線**：brand-web 預設連 `qrepjm4m`。若要用環境變數覆寫，複製 `.env.example` 成 `.env`：
   ```env
   VITE_SANITY_PROJECT_ID=qrepjm4m
   VITE_SANITY_DATASET=production
   VITE_SANITY_API_VERSION=2024-10-01
   ```
2. **建立順序（重要）**：先建「服務項目 / 方案」這類獨立文件，再到「首頁」用參考（reference）把它們挑進來。
   建議順序：① 服務項目 → ② 方案 → ③ 最新消息 → ④ 網站設定 → ⑤ 首頁。
   （最新消息為獨立文件，前端直接抓取，不需在首頁挑選，但同樣要 Publish。）
3. **多語欄位**：型別為「多語字串 / 多語長文字」者，請至少填 **中文（zhTW）**。前端找不到目前語言會自動退回中文。
4. **每筆都要按 Publish**：前端 client 設定 `perspective: 'published'`，**草稿不會顯示**。
5. **CDN 快取**：`useCdn: true`，發布後若沒立即更新，等幾分鐘或硬重新整理。

## 3.5 最新消息 `news`（清單 · 建議先建 3 筆）

對應畫面：首頁「最新消息」區（位於方案介紹下方）。前端會**置頂優先、再依發布日期新到舊取前 3 筆**。

> 前端卡片用到：`title`（標題）、`category`（分類徽章）、`publishedAt`（日期）、`excerpt`（摘要）、`coverImage`（封面圖，沒填則顯示「-」佔位圖）。
> `category` 為下拉選項：公告 `announcement` / 活動 `event` / 媒體報導 `press`。
> `slug`、`publishedAt` 為必填；`isPinned` 開啟可固定排在最前。`body`（內文）目前前端首頁卡片未使用，可先不填。

| # | 標題 `title` | 分類 `category` | 摘要 `excerpt` | 置頂 `isPinned` |
|---|---|---|---|---|
| 1 | `守護健檢會員平台正式上線` | 公告 | `一站式健檢會員服務正式啟用，立即註冊即可線上預約合作診所、數位化管理健檢報告。` | **開啟** |
| 2 | `新春健檢專案開跑` | 活動 | `即日起完成健檢預約即享好禮，活動期間加購家庭子帳號再享優惠。` | 關閉 |
| 3 | `媒體報導：數位健檢成家庭照護新趨勢` | 媒體報導 | `多家媒體報導本平台如何以數位化服務，協助家庭長期管理全家人的健康。` | 關閉 |

> `publishedAt`（發布日期）請各自挑一個日期；要呈現「新到舊」就讓第 1 筆日期最新。

## 6. 建好後如何驗證

1. 後台每筆文件按 **Publish**。
2. 啟動前端：`cd brand-web && npm run dev`，開啟終端機顯示的網址。
3. 已填欄位會帶出內容；未填（或第 5 節無欄位者）顯示「-」。
4. 若沒更新：確認已發布、`projectId` 是否為 `qrepjm4m`、稍候 CDN 快取。
